### 📌 Огляд базових факторів ціни

Для точної оцінки впливу окремих характеристик на вартість оренди застосовувався **метод стратифікації**. Цей аналітичний підхід передбачає розподіл загального масиву даних на вузькі, однорідні підгрупи (страти) за ключовими параметрами. Це дозволяє ізолювати досліджуваний фактор та розрахувати його «чисту» ціну, мінімізуючи спотворення від інших змінних.

**Ключові інсайти:**

* 📉 **Парадокс метражу:** Однокімнатні квартири мають найвищу вартість квадратного метра (361 грн/м²).
* 📐 **Формат студії:** Планування «кухня-студія» підвищує медіанну ціну 1-кімнатної квартири майже на 42%.
* 🚇 **Фактор метро в Києві:** Житло в пішій доступності до метро в середньому на 7 000 грн дорожче.
* 🏢 **Поверховість:** Квартири на високих поверхах (9+) стабільно залишаються найдорожчими в усіх великих містах.
* 📊 **Структура ринку:** Абсолютна більшість пропозицій (87%) зосереджена в сегменті 1- та 2-кімнатних квартир.

### 📌 Огляд сучасних факторів ціни

Для комплексної оцінки впливу різних характеристик на ціну додатково застосовано **матрицю кореляцій**. Цей інструмент дозволяє математично виміряти силу зв'язку між кожним окремим фактором та фінальною вартістю оренди. Панель фокусується на «кризових» характеристиках, показуючи їхню реальну вагу на ринку порівняно з традиційними метриками.

**Ключові інсайти:**

* 🐾 **Pet-friendly статус:** Дефіцит лояльних до тварин орендодавців (лише 18% пропозицій) формує націнку в розмірі 3 000 грн порівняно з житлом "без тварин".
* ⚡ **Пріоритети автономності:** Найдорожчою опцією під час блекаутів є працюючий ліфт (надбавка 3 000 грн), тоді як базові потреби (вода, опалення) оцінюються ринком значно дешевше.
* 🛡️ **Географія безпеки:** Переплата за наявність укриття сильно залежить від регіону і досягає своїх максимумів у Києві (2 500 грн) та Дніпрі (2 125 грн).
* 📊 **Матриця кореляції:** Традиційні параметри (площа об'єкта) продовжують визначати ціноутворення значно сильніше (0,62), ніж сучасні фактори безпеки чи автономності (0,15–0,29).


## 🛠 Методологія та технічний стек

**Стек:** Python (Selenium, BeautifulSoup, Undetected_chromedriver, Cloudscraper, Time, Pandas, Re, Matplotlib, Seaborn) | Power BI (Power Query, DAX)

<details>
<summary><b>1. Збір даних</b></summary>
<br>
Через жорсткі ліміти офіційного API було розроблено власний парсер на Selenium для сайту DOM.RIA. Зібрано понад 10 000 оголошень.
<ul>
  <li><b>Обхід захисту:</b> Використано <code>cloudscraper</code> для первинних запитів та <code>undetected_chromedriver</code> для взаємодії з динамічними елементами.</li>
  <li><b>Відмовостійкість:</b> Впроваджено власні функції-обгортки (<code>safe_find</code>) з блоками <code>try-except</code> для запобігання падінню скрипта при відсутніх даних.</li>
  <li><b>Збереження:</b> Головний цикл загорнутий у <code>try...finally</code>, що гарантує збереження зібраного масиву у CSV навіть при примусовій зупинці парсера.</li>
</ul>
</details>

</ul>
